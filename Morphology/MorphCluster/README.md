# Clustering of the Morphological Data

These scripts are used to cluster specifically the morphological data of the MicroBioVision-programm and furthermore specifically for the data generated for my master thesis. Though it can be used as a basis for other sets of data. Mainly the folder structure is the issue which will be explained further down this README. If there are any questions on how to change the code to adapt for different folder structures, do not hesitate to contact me.

## Code Structure
The code is divided in four parts. This is on the one hand due to the specific structure of the data and on the other hand that it leaves the human to decide on the number of clusters. The data it is used to can be found in the Input folder. There you have 27 folders with three subfolders each. Each main folder is one cultivation condition and each subfolder is for each sampling day (3, 5 and 7 days of cultivation). In each subfolders are several excel files gathered from the MicroBioVision code. This one translated each microscopic images to one excel file with several information about each pellet detected (diameter, perimeter, circulation, compactness, etc.). Subsequently it is described what each code does:

### One - Simplification
The simplest code. For each cultivation and each day it gathers the lists in the excel files and creates on excel file each with the collected infromation. This is the saved in the output folder, which mimics the structure of the input folder.

### Two - Preparing the Clustering, Performing Clustering, and Plotting PSD for Overview

This script prepares the data for clustering, performs the actual clustering, and generates visualizations for quick overview. It walks through each subfolder in the Output directory, looking for the combined_data.csv files generated by the first script. 
For each file, it prepares and visualises the data by:
- Cleaning the folder by removing old results, keeping only the input data to prevent stacking of outdated files.
- Assigning histogram bins to each pellet’s diameter using linear binning (25 bins by default). This allows grouping pellets by size range.
- Saving a clean PSD file (PSD_histogram_input.csv) with explicit bin assignments, volume, and volume fraction for each pellet.
- Aggregating the data into histogram bins, calculating both number-weighted and volume-weighted particle size distributions (PSD). This is saved as PSD_histogram_bins.csv.
- Generating a volume-weighted PSD plot (PSD_volume_weighted.png) for quick visualization.

Clustering:
The script then performs clustering on the prepared data, using two distinct approaches for each possible number of clusters (k=2 to 7):
- Volume Fraction Clustering: Clusters pellets based on their volume fraction. This approach groups pellets according to their contribution to the total volume, which is useful for identifying dominant size classes by volume.
- Diameter Frequency Clustering: Clusters pellets based on their frequency of occurrence (i.e., their index in the dataset). This approach groups pellets according to their distribution in the dataset, which is useful for identifying common size classes by count. Though this also translates to equally sized clusters with no regard to their size, as each pellet size appears once and the distance between the pellet sizes are thus stripped back to one.

For each clustering approach and each value of k, the script assigns cluster labels to each pellet and saves all results in a single file (PSD_clustered_summary.csv). This file contains the original data along with all cluster assignments, making it easy to analyze and compare different clustering configurations.


### Three - Cluster Validation and Visualization

This script evaluates the quality of the clustering results generated in the previous step and provides visualizations to help decide the optimal number of clusters. It is designed to be run after the preparation and clustering scripts, as it relies on the PSD_clustered_summary.csv files generated in each subfolder.
The script performs the following steps:
- Cluster Validation: It iterates through all PSD_clustered_summary.csv files in the Output directory. For each file, it calculates two key metrics for each clustering approach (Volume Fraction and Number-based) and for each possible number of clusters (k=2 to 7):
    - Silhouette Score: Measures how similar a pellet is to its own cluster compared to other clusters. Higher values indicate better-defined clusters.
    - RMSSTD (Root Mean Square Standard Deviation): Assesses the compactness of clusters. Lower values indicate tighter, more cohesive clusters.
- Aggregation and Saving: The results for all files are aggregated and saved in a single CSV file (Evaluation_Results.csv) in the Evaluation Clustering folder, which is created at the same level as the Output directory. This file contains the mean silhouette score and RMSSTD for each clustering method and each value of k, as well as the number of files included in the evaluation.
- Visualization: The script generates four plots to visualize the evaluation results:
    - Silhouette Score vs. Cluster Count for both Volume Fraction and Number-based clustering.
    - RMSSTD vs. Cluster Count for both Volume Fraction and Number-based clustering.
    - These plots are saved as PNG files in the Evaluation Clustering folder and help identify the optimal number of clusters by showing how the metrics change with increasing k.

Now it is your job: Decide on the right clustering technique and what number of clusters seems to be the best (have a look in my master thesis if you are unsure about the evaluation)


### Four – Final Evaluation and Visualization of Clusters

This script performs the final evaluation of the clustered data, calculating key statistics for each cluster and generating visualizations. It is designed to be run after the clustering process is complete, using the PSD_clustered_summary.csv files generated in the previous step.
Key Features:
- Manual Input Required: The user must specify the clustering type (Volume or Number) and the number of clusters (k) to evaluate. This allows for flexible analysis based on the results from the previous clustering evaluation step.
- Cluster Statistics: For each folder containing clustered data, the script calculates and saves the following statistics for each cluster:
    - Mean and standard deviation of pellet diameter
    - Total volume fraction
    - Mean and standard deviation of circularity (irregularity)
    - Mean and standard deviation of compactness (diameter ratio)
    - Number of particles in each cluster
- Output: The results are saved as a CSV file (PSD_cluster_summary_final_k{cluster_count}_{cluster_type}.csv) in each subfolder, providing a concise summary of cluster properties.

Visualization:
- The script also generates two plots for each dataset:
    - Volume-weighted Particle Size Distribution (PSD): A histogram showing the distribution of pellet diameters, with cluster boundaries marked as dashed lines. The plot includes a smoothed curve for better visualization of trends.
    - Cumulative Volume-weighted PSD: A cumulative distribution plot, again with cluster boundaries marked, to visualize how volume fraction accumulates across pellet sizes.
